Agent-Based System for Centralized File Deletion

A coordinated platform where a central admin service dispatches file scanning and deletion tasks to lab agents, with a verification-first approval process.

Explore Project Deploy on Vercel

Overview

This system helps administrators manage unsafe or outdated files across multiple lab machines from one control point. It combines command dispatching, client status tracking, file preview, and approval-gated deletion to reduce accidental data loss.

Core Problem

Manual cleanup across many computers is slow, inconsistent, and hard to audit.

Solution

A networked multi-agent system with admin verification before destructive actions.

Architecture

Backend (Master)

Handles orchestration, task dispatch, agent registry, and result collection.

Client Agent

Scans target files, reports findings, and executes deletion only when approved.

Admin UI

Flask-based dashboard for submitting instructions and approving file operations.

Operational Workflow

  1. 1 Admin submits a natural-language deletion instruction.
  2. 2 Backend distributes scan tasks to online agents.
  3. 3 Agents return candidate files for review.
  4. 4 Admin approves or rejects pending file actions.
  5. 5 Approved files are deleted and actions are logged.

Technology Stack

Python Flask SQLAlchemy SQLite TCP Networking Docker HTML/CSS/JavaScript
